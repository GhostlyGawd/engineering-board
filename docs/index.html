Why it's different

Agents now do real multi-session work — and they forget everything between sessions and trip over each other in parallel. The field still forces a choice: a board that's visible but dumb, or memory and coordination that's smart but opaque — kept in a database, or in ~/.claude outside your repo. engineering-board refuses the trade-off.

Visible, diffable state

Your agents' board is committed markdown — cards, an index, a graph — reviewed in the same PRs as code. The board is the database.

Durable memory

Your agent forgets everything between sessions. Recurring lessons promote into committed Learning entries (L###) that survive — surfaced at session start and in a dedicated Learnings panel on the board view.

Collision-free agents

Two agents editing the same file clobber each other. Atomic claim-locking lets parallel worker agents run without stepping on each other.

Why not Claude Code's built-in Tasks?

Use both. Native Tasks are good personal tracking — they persist across sessions, support dependencies, and come with a Ctrl+T board. But they live in ~/.claude/tasks/, per-user and per-machine, outside the repo: invisible in PRs, invisible to teammates, with no capture pipeline, no review states, no committed learnings. engineering-board is the repo's shared, reviewable board. Keep native Tasks for in-session personal tracking; the board holds the project's durable, team-visible state.

Install both ways

Free & MIT. Requires Claude Code (also free). One shared board format, two thin adapters.

Claude Code plugin · start here

/plugin marketplace add GhostlyGawd/engineering-board
/plugin install engineering-board

# then, in a project:
/board-init my-project

Installing the plugin also registers the MCP server via the bundled .mcp.json.

MCP server (any client)

# clone the repo, then point python3 at the server:
git clone https://github.com/GhostlyGawd/engineering-board

# Claude Code
claude mcp add engineering-board -- \
  python3 "$(pwd)/engineering-board/mcp-server/engineering_board_mcp.py"

# Claude Desktop: claude_desktop_config.json
"mcpServers": {
  "engineering-board": {
    "command": "python3",
    "args": ["/path/to/mcp-server/engineering_board_mcp.py"]
  }
}

Pure python3, zero dependencies. 11 tools over stdio.

Uninstall any time — the board stays as plain markdown in your repo, so there's no lock-in. And it's built for untrusted input: scratch content is treated as data, never instructions, and hardened against prompt injection (security posture).
