# CLAUDE.md

> Auto-loaded every session. Keep it tiny — it's a pointer, not a manual.

**Start here:** read **[`state.md`](state.md)** first. It's the cross-session handoff
file — current snapshot, active branch, decisions (with dates), what's done, what's
next, and the "any session" repo invariants. Each web session starts from a **fresh
clone**, so `state.md` is how you load context fast.

**Update as you go — not just at the end:** when you push, finish a thread, or settle a
decision, refresh the relevant sections of `state.md` (and its `_Last updated:_` line).
A session can stop abruptly, so don't bank it all for a wrap-up that may never come.
Keep it high-signal — a pointer to context, not a log.
