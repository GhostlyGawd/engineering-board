## 2. Findings (ranked by attention reclaimed per line changed)

### F1 — Install: the recommended path doesn't win the squint · **Medium**
Both install cards are the same width, and the heaviest element in each is a
dark `<pre>` block sharing one rule — `pre{background:var(--eb-ink);color:#ECEBE6;…}`
(`docs/index.html:155`). Two equal black rectangles read as **two equal
options**. The only thing marking the plugin path as primary is a 3px accent
left-border (`.install .card2.recommended`, `:153`) and an eyebrow reading
"start here" (`:253`) — both are hairline signals that lose to the mass of the
code block beside them. A first-time visitor's eye is split where the page most
wants to say "do this one."

### F2 — Priority pills are redundant chrome for the majority of cards · **Medium**
`.prio.p0` (danger fill) and `.prio.p1` (accent fill) now carry real weight
(`board-view.sh:317–318`) — good. But P2 and P3 have no rule of their own, so
they fall through to the base `.prio{color:muted;border:1px solid border}`
(`:315`): an identical grey outline pill. On the eb-self board **every open
card is P3** (`board.html` To-do column) — so the pill repeats the same grey
"P3" on card after card, spending a chip of chrome per card to say nothing that
distinguishes it. Lens 7 (signal-to-chrome): a badge that never varies is
decoration, not signal.

### F3 — The board's organizing labels are its faintest text · **Low**
The four column headers name the pipeline — the product's core mental model —
yet `.col-h` is `.7rem` (≈11px) uppercase in `--eb-text-muted` (`:298–299`),
the smallest, lowest-contrast type on the surface. The per-lane `.count`
(`:300`) — the fastest "where is the work piled?" signal (6 / 0 / 0 / 58) —
inherits that same faint treatment. Meanwhile the Learnings heading was
(correctly) promoted to the second-largest text on the page (`:325`). Net: the
one intentional promotion slightly out-weighs the labels that structure the
actual work. Open cards still win via opacity, and Learnings sits below the
column band, so this is a residual tilt, not a wrong winner.

### F4 — Empty active lanes cede half the board's width to dead space · **Low** (see LAYOUT)
`.cols{grid-template-columns:repeat(4,1fr)}` (`:289`) fixes four equal columns
regardless of fill. When work clusters in To-do + Done (the common shape), the
Review and Validate columns render as two near-empty 25%-width panels showing
only `.empty{—}` (`:321`), while the collapsed-but-still-10-card Done column
carries the most mass on the board. This is primarily a **layout** call — noted
here because it shapes the squint; the fix belongs in `LAYOUT.md` (stage 57).

## 3. Fixes (subtraction first — quiet what shouts before promoting anything)

1. **F1 — demote the alternative, don't inflate the recommended.** Keep the
   plugin card's dark `<pre>`; give the MCP card's block a lighter, recessive
   treatment (e.g. `--eb-surface` background + muted border instead of the ink
   fill) so the recommended path's code block wins by contrast, not by adding
   weight. One new rule scoped to `.install .card2:not(.recommended) pre`.
2. **F2 — remove the P3 pill entirely; it is the floor, not a rank.** Render
   the pill only for P0–P2 (or only P0/P1). A card with no pill *is* the "P3"
   signal by absence. Deletes chrome from the majority of cards; the urgent
   fills stand out more once the grey pills stop competing.
3. **F3 (optional) — give `.count` the one step of weight it earns.** It is the
   scannable triage signal; a single `font-weight:600` + `--eb-text` on
   `.count` (`:300`) lets the pile-up read at a glance without touching the
   label. This is a promotion, so treat it as optional — the alternative is to
   leave it and accept the restraint.
4. **F4 — defer to `LAYOUT.md`.** Track it there (fluid/collapsing columns),
   not as a hierarchy change.

Estimated attention reclaimed: F1 fixes the one genuine split-focus on the
landing page; F2 declutters every card on a real board. Both are ~1 rule.

## 4. The one rule this codebase should adopt

> **One signal per emphasis — and only the primary gets a fill.**
> Size carries heading hierarchy; a single solid accent (or danger) **fill**
> marks the *one* primary per screen — the Install button, the "us" row, a P0
> card; **opacity** (not deletion) carries de-emphasis. Everything else —
> metadata, counts, secondary paths, floor-priority badges — stays in the
> muted-outline register. An element earns emphasis by taking **exactly one**
> step up the scale, never two, and a supporting element never borrows the
> primary's fill or renders a badge that never varies.

The tokens and both surfaces already mostly obey this; the residuals above are
the two places they don't (a secondary path that fails to *claim* its one
signal — F1; a badge that renders when it has nothing to say — F2).
