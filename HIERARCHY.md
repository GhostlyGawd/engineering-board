# Visual Hierarchy Audit

> **Status:** Fixes 1–7 were applied in v1.5.1; Fix 8 (the token drift guard)
> landed after and is currently unreleased — see the CHANGELOG. The screen table
> and findings describe the **pre-fix** state (the "actually wins" column is what
> the audit found); each fix in §3 is annotated with its outcome.

Does each key screen direct attention to what the product needs seen? This audit
ranks every screen's elements by the visual weight the **actual CSS** assigns,
then asks whether the intended focal point wins. Read-only pass; the only write
is this file.

**Surfaces & styling sources.** The product has exactly two rendered surfaces,
each a self-contained HTML document with its design tokens **inlined** (not
linked):

- **Landing page** — `docs/index.html` (`<style>` at lines 30–158; tokens 32–58).
- **Board view** — generated by `hooks/scripts/board-view.sh` (`<style>` heredoc
  lines 262–337); the committed example is `engineering-board/eb-self/board.html`.
- **Token "single source"** — `brand/tokens.css` (mirrored, lossily, into both of
  the above and into `docs/assets/tokens.css`). See Finding 7 — the copies drift.

## 2. Findings (ranked by attention reclaimed per line changed)

### F1 — Board priority has data but no visual weight *(highest leverage)*
Cards emit `class="prio p0"`, `p1`, `p2`, `p3` (see `board.html:85` — a P0 "MCP
path traversal … arbitrary rm -rf" and a P3 "cosmetic" both appear). But the
stylesheet defines **only** `.prio` (`board-view.sh:310–311`):

```css
.prio{...color:var(--eb-accent-cur);border:1px solid var(--eb-accent-cur)}
```

There is **no `.prio.p0` / `.p1` / `.p2` / `.p3` rule anywhere.** Every severity
renders as the *same* ~9.9px (`.62rem`) amber outline. The board's single most
important signal — how bad is this — is encoded in the markup and thrown away in
the CSS. Squint at the board and P0 and P3 are indistinguishable.

### F2 — The Done column dominates by mass; open work loses
Layout is four equal columns (`.cols{grid-template-columns:repeat(4,1fr)}`,
`board-view.sh:289`). On the live board that is **6 open vs 58 done**
(`board.html:85`): "Review 0" and "Validate 0" render as tall empty "—" columns,
while "Done" shows 10 full cards then `+48 more`. The densest, tallest region of
the screen is the *least* actionable content (already-resolved work). Every Done
card carries the same `border` + `box-shadow` as an open card
(`.card`, `board-view.sh:296–297`), so finished work competes at full weight
with what needs doing.

### F3 — The Learnings "moat" is styled as a footnote
`board-view.sh` comments call Learnings "the moat … the viewer highlights them
rather than burying them" (lines 120–123). The styling does the opposite:

- Section header `.lane-h` is `.8rem` (12.8px), uppercase, `--eb-text-muted`
  (`board-view.sh:315`) — **identical** to the "Questions · Observations" header.
  The differentiator and the miscellany get the same billing.
- It renders **dead last**, after all 58 Done cards and the `<details>` fold
  (`board-view.sh:203–234` assembles `learn_html` after the columns).
- Its one emphasis is a `border-left:3px solid var(--eb-accent-cur)`
  (`board-view.sh:320`) on `.82rem` text — a whisper under a wall of Done.

The whole landing page sells "durable memory"; the actual product view files it
at the bottom in 12–13px muted type.

### F4 — Hero: the decorative demo outshouts the conversion action
The intended action is **Install** — a `.btn-primary` at `--eb-fs-sm` **14px**
with a `--eb-accent` fill (`index.html:96–97`). Directly below it sits `.demo`, a
60rem bordered/shadowed panel whose `.card.live` element has an **amber glow**
`box-shadow:0 4px 14px rgba(154,91,0,.22)`, an accent border, an amber dot
(`index.html:115–117`), **and continuous motion** (`@keyframes settle`,
`index.html:119–120`). Motion + saturated glow beat a small static pill every
time. The one saturated animated thing on the entry screen is an illustration,
not the CTA. (The demo also carries its own `▸ Play` button — a second
interactive control in the hero.)

### F5 — "Why" is five co-equal cards: everything competes, nothing wins
`.vp` renders five `.card2` with identical surface/border/radius/padding and
`h3` all at `--eb-fs-md` 18px (`index.html:128–130`). Importance is flat: the
lead claim ("Visible, diffable state", `index.html:207`) is styled exactly like
"Runs where you are" (`index.html:211`). No card is a hero; the section has no
reading anchor. This is the textbook "nothing wins" state.

### F6 — Accent overload: amber can no longer mark importance
One hue (`--eb-accent` / `--eb-accent-dark`) is spent on **~8 unrelated jobs**:
links (`index.html:67`), the eyebrow label (`index.html:72`), the primary button
fill (`index.html:97`), the demo glow (`index.html:115`), every "Yes" in the
compare table (`index.html:139`), **all** priority pills (`board-view.sh:311`),
learning-confidence badges (`board-view.sh:324`), the "kind" label
(`board-view.sh:318`), and the Learnings left-border (`board-view.sh:320`).
When the accent means "link" *and* "primary" *and* "P0" *and* "decoration," it
means nothing — it can't reserve contrast for what actually matters (F1, F4).

### F7 — Emphasis is defined in 3–4 drifting copies (consistency risk)
`brand/tokens.css` is billed as the "single source of truth" (its header,
`brand/tokens.css:2`), but the tokens are **duplicated** — inlined into
`index.html` (32–58, comment "mirror of brand/tokens.css"), inlined again into
`board-view.sh` (263–281), and copied into `docs/assets/tokens.css` (which
`index.html` does not even link). They have **already drifted**: `brand/tokens.css`
defines `--eb-card` and `--eb-danger`; `docs/assets/tokens.css` omits both
(`diff` confirmed). Nothing stops one surface's hierarchy from diverging from
another's.

### F8 — Install offers two primaries (so, zero)
`.install{grid-template-columns:1fr 1fr}` (`index.html:144`) with two identical
`.card2`. For a first-timer the plugin path is the default and MCP is advanced,
but the CSS gives them equal weight — the user must read both to learn which is
"the" way in. "Two primaries is zero primaries," applied to onboarding.

## 3. Fixes (demotions first — the fastest hierarchy fix is quieting what shouts)

Ranked by attention reclaimed per line of CSS.

1. **Encode board priority; demote P2/P3 (F1, F6).** ✅ **Applied.** `.prio` base
   is now a neutral muted outline (`--eb-text-muted` / `--eb-border`); `.prio.p0`
   is a filled `--eb-danger` chip and `.prio.p1` a solid accent chip
   (`board-view.sh`). Severity is legible at a squint; low-priority pills no
   longer borrow the accent.
2. **Recede the Done column (F2).** ✅ **Applied.** Columns now carry a
   `col-<key>` class; `.col-done .card{opacity:.6;box-shadow:none}` dims resolved
   work (restored on hover, unchanged in print). The finished pile stops
   out-massing open work. (`DONE_VISIBLE` left at 10.)
3. **Quiet the hero demo glow (F4).** ✅ **Applied.** `.card.live` `box-shadow`
   changed from the amber glow to the quiet `var(--eb-shadow-card)` (both light
   and dark), keeping only the thin accent border.
4. **Promote the Install CTA (F4).** ✅ **Applied.** `.btn-primary` is now
   `--eb-fs-base` with `padding:.85rem 1.5rem`; the flex row's `stretch` keeps
   the ghost button aligned to its height, so the primary out-weighs both it and
   the demo's Play.
5. **Give Learnings a real heading (F3).** ✅ **Applied.** A `.lane-h-learn`
   modifier gives the "Learnings · durable memory" heading `1.05rem`,
   full-contrast `--eb-text`, sentence case. Combined with the Done-recede, the
   moat reads as a section. (Kept its position below the active columns —
   promoting it *above* To-do would bury actionable work.)
6. **Break the "Why" grid (F5).** ✅ **Applied.** The lead "Visible, diffable
   state" `.card2.lead` spans two columns with a larger `h3`/body, anchoring the
   section; collapses to a single column under 520px.
7. **Mark the plugin install as primary (F8).** ✅ **Applied.** The plugin
   `.card2.recommended` gets an accent left-border and a "start here" eyebrow;
   the MCP path stays quiet.
8. **Collapse the token copies to one (F7).** ✅ **Applied** (post-v1.5.1) — as a
   *drift guard*, not codegen. Fully generating the inline blocks from a single
   source needs a build step, which conflicts with the deliberate zero-dependency,
   committed-and-byte-deterministic design of these surfaces. So instead of
   forcing a build step: the dead, unreferenced third copy
   (`docs/assets/tokens.css` — which had already dropped `--eb-card` /
   `--eb-danger`) is deleted, and a new `tests/token-coherence.sh` fails CI if any
   token value a surface mirror inlines diverges from `brand/tokens.css`
   (font-stack fallbacks, which surfaces legitimately trim, are exempt). The
   source of truth is now *enforced* rather than merely *labelled*, with no build
   step added.

## 4. The one rule this codebase should adopt

> **One accent, one job per screen.** The amber accent marks the *single* most
> important thing on a screen — the primary action, or the highest-severity
> signal — and nothing else. Encode importance in **size, weight, and color**,
> not in borders and boxes. When everything is amber-bordered and 14px, promote
> nothing: subtract until one thing wins.

Every finding above is one violation of that rule: severity that isn't colored
(F1), finished work as loud as open work (F2), the moat in muted 12px (F3), a
decoration glowing louder than the CTA (F4), five co-equal cards (F5), an accent
spread across eight jobs (F6).
